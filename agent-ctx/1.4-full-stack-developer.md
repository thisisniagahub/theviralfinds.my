### Pull-to-Refresh
- `src/hooks/use-pull-to-refresh.ts` — touch-only hook (gated by `matchMedia('(pointer: coarse)')`):
  - Triggers only when `window.scrollY === 0`
  - Dampened rubber-band pull (0.5 resistance, 90px max, 70px threshold)
  - `preventDefault` on touchmove only when actively pulling (so normal scroll-to-top works)
  - Returns `{ pullDistance, isRefreshing, isPulling, progress }`
- `src/components/pwa/pull-to-refresh-indicator.tsx` — circular spinner that rotates with pull progress, arrow flips when threshold reached, switches to spinning RefreshCw when refreshing
- `src/components/pwa/pull-to-refresh-wrapper.tsx` — wraps page content, translates by `pullDistance`, mounts indicator above content
- Integrated in `src/app/page.tsx` — wraps `<PageComponent />`, refresh callback calls `queryClient.invalidateQueries()` + toast

### Mobile-specific CSS (globals.css)
- `html`: `-webkit-text-size-adjust: 100%`, `scroll-behavior: smooth`, antialiased font rendering
- `body`: `overscroll-behavior-y: none`, `-webkit-overflow-scrolling: touch`, `-webkit-tap-highlight-color: transparent`, `text-rendering: optimizeLegibility`, `-webkit-touch-callout: none` (UI chrome only)
- Force `font-size: 16px` on inputs below sm breakpoint → prevents iOS auto-zoom-on-focus
- Re-enabled `user-select: text` on text-bearing elements
- `.safe-area-inset-top/bottom/left/right/` utilities (env(safe-area-inset-*))
- Mobile `@media (max-width: 1023px)`: `main` padding-bottom `5rem + env(safe-area-inset-bottom)` to reserve space for mobile bottom nav
- `.scroll-container { overscroll-behavior: contain; }` to prevent PTR interference inside scrollable areas
- `@media all and (display-mode: standalone)`: hide scrollbars for native PWA feel
- `:focus-visible` outline (2px shopee orange) for keyboard nav; `:focus:not(:focus-visible)` hidden (no focus ring on tap)
- `button, a, [role="button"/"tab"/"menuitem"] { touch-action: manipulation; }` — removes 300ms tap delay on older iOS
- `@media (hover: none)`: `:active` scale 0.97 for touch feedback
- `img[loading="lazy"]` opacity transition for smooth fade-in
